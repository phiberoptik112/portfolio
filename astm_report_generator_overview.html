GUI Component

The graphical user interface is built using Kivy and consists of:

  • Test Plan input file - needs to be properly formatted for current MLP version
  • Raw SLM data path input
  • Load data - Uses the FileLoader class to load the data from the GUI into a test_data_collection dictionary
  • Output all reports - Uses the DataProcessor class to process the data from the test_data_collection dictionary into all the aspects of the Reportlab functions.

This beigns with a FileLoader class that loads the input data from the GUI, does a bit of cleanup, and then passes the data to the DataProcessor class.

Data Processor

The data processing backend utilizes pandas and numpy for efficient calculations:

  • Sound level meter data parsing and validation
  • Apparent Tansmission Loss, Noise Reduction calculations per ASTM standards
  • Data normalization and correction factors application
  • ASTC, AIIC, NIC, DTC Curve generation and plotting elements for the report

This section has become an area to place verification and validation methods to trace if there has been issues with loading the proper files and their data.

Report Builder

Report generation is handled through ReportLab formatting and matplotlib for the charts and graphs:

  • PDF generation with consistent formatting
  • Table and graph creation using matplotlib
  • Table generation for measurement data
  • Automated placement of results and analysis
  • Custom header/footer implementation

The report builder maintains compliance with ASTM documentation requirements, and displays any errors or warnings given the final calculations and results.
